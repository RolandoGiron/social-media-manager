# Plan 07-01: n8n Appointment Reminders Workflow (CAL-03)

**Phase:** 07 — Automation Layer + Dashboard  
**Plan:** 01 of 03  
**Goal:** Create the `appointment-reminders` n8n cron workflow that fires every 15 minutes, detects appointments in the 24h and 1h reminder windows, renders a message from the `recordatorio` template, sends via WhatsApp, and marks the reminder flags as sent. Failures are retried once and logged to `workflow_errors`.

**Requirements covered:** CAL-03  
**Success criteria addressed:** SC-1 — *Patient with a booked appointment receives a WhatsApp reminder automatically 24 hours before and again 1 hour before, without admin action*

### Decisions from 07-CONTEXT.md
- **D-01:** Template: `SELECT body, variables FROM message_templates WHERE category = 'recordatorio' AND is_active = true LIMIT 1`
- **D-02:** Variables injected: `{{nombre}}` (patient first_name), `{{tipo}}` (appointment_type), `{{fecha}}` (date formatted), `{{hora}}` (time formatted). Clinic data embedded in template body.
- **D-03:** Timezone: `America/Mexico_City` (env var `TZ`). Format: "lunes 20 de abril" + "10:00 AM"
- **D-04:** Failure: 1 retry at 30s delay. On second failure: log to `workflow_errors` + set `reminder_Xh_sent = true` (prevent infinite retries)
- **D-05:** Cron: every 15 minutes. 24h window: `scheduled_at BETWEEN now() + INTERVAL '23h45m' AND now() + INTERVAL '24h15m'`. 1h window: `scheduled_at BETWEEN now() + INTERVAL '45m' AND now() + INTERVAL '1h15m'`

Structure:
```
[Schedule Trigger] → [Query 24h Appointments] → [Loop Over 24h Items] → [Send 24h Reminder] → [Mark 24h Sent]
                   ↘ [Query 1h Appointments]  → [Loop Over 1h Items]  → [Send 1h Reminder]  → [Mark 1h Sent]
```

Workflow metadata:
```json

  "name": "appointment-reminders",
  "active": false,
  "tags": ["automation", "appointments"]
```

### Task 2: Schedule Trigger node
**Node name:** `Schedule Trigger`  
**Type:** `n8n-nodes-base.scheduleTrigger`

Configuration:
```json
{
  "rule": {
    "interval": [
      {
        "field": "minutes",
        "minutesInterval": 15
      }
    ]
  }
}
```

### Task 3: Query 24h appointments node
**Node name:** `Query 24h Appointments`  
**Type:** `n8n-nodes-base.postgres`  
**Operation:** executeQuery

SQL:
```sql
SELECT
    a.id AS appointment_id,
    a.scheduled_at,
    a.appointment_type,
    p.first_name AS nombre,
    p.phone_normalized AS phone
FROM appointments a
JOIN patients p ON a.patient_id = p.id
WHERE a.status = 'confirmed'
  AND a.reminder_24h_sent = false
  AND a.scheduled_at BETWEEN NOW() + INTERVAL '23 hours 45 minutes'
                         AND NOW() + INTERVAL '24 hours 15 minutes'
```

### Task 4: Query 1h appointments node
**Node name:** `Query 1h Appointments`  
**Type:** `n8n-nodes-base.postgres`  
**Operation:** executeQuery

SQL:
```sql
SELECT
    a.id AS appointment_id,
    a.scheduled_at,
    a.appointment_type,
    p.first_name AS nombre,
    p.phone_normalized AS phone
FROM appointments a
JOIN patients p ON a.patient_id = p.id
WHERE a.status = 'confirmed'
  AND a.reminder_1h_sent = false
  AND a.scheduled_at BETWEEN NOW() + INTERVAL '45 minutes'
                         AND NOW() + INTERVAL '1 hour 15 minutes'
```

### Task 5: Fetch reminder template node (shared)
**Node name:** `Fetch Reminder Template`  
**Type:** `n8n-nodes-base.postgres`  
**Operation:** executeQuery

SQL:
```sql
SELECT body, variables
FROM message_templates
WHERE category = 'recordatorio'
  AND is_active = true
ORDER BY created_at ASC
LIMIT 1
```

Connect after Schedule Trigger (runs once per cron tick, before both loops).

**No-template guard:** Add an `If` node after this query — if result is empty, stop execution and log a `workflow_errors` entry: `{ workflow_name: 'appointment-reminders', error_message: 'No active template with category=recordatorio found' }`.

### Task 6: Loop over 24h items + render message
**Node name:** `Loop 24h Items`  
**Type:** `n8n-nodes-base.splitInBatches`  
**Batch size:** 1

Inside the loop, add a **Function node** named `Render 24h Message`:
```javascript
const item = $input.first().json;
const template = $('Fetch Reminder Template').first().json;

// Format date/time in America/Mexico_City
const tz = 'America/Mexico_City';
const dt = new Date(item.scheduled_at);

const DAYS = ['domingo','lunes','martes','miércoles','jueves','viernes','sábado'];
const MONTHS = ['enero','febrero','marzo','abril','mayo','junio',
                'julio','agosto','septiembre','octubre','noviembre','diciembre'];

const dtMX = new Intl.DateTimeFormat('es-MX', {
  timeZone: tz,
  weekday: 'long',
  day: 'numeric',
  month: 'long',
  hour: '2-digit',
  minute: '2-digit',
  hour12: true
});

const parts = dtMX.formatToParts(dt);
const get = (type) => parts.find(p => p.type === type)?.value ?? '';

const fecha = `${get('weekday')} ${get('day')} de ${get('month')}`;
const hora = `${get('hour')}:${get('minute')} ${get('dayPeriod')}`;

let body = template.body
  .replace(/\{\{nombre\}\}/gi, item.nombre || '')
  .replace(/\{\{tipo\}\}/gi, item.appointment_type || '')
  .replace(/\{\{fecha\}\}/gi, fecha)
  .replace(/\{\{hora\}\}/gi, hora);

return [{ json: { ...item, rendered_message: body } }];
```

### Task 7: Send 24h reminder via sub-workflow
**Node name:** `Send 24h WA Reminder`  
**Type:** `n8n-nodes-base.executeWorkflow`  
**Source:** database  
**Workflow name:** `sub-send-wa-message`

Parameters passed:
- `phone`: `{{ $json.phone }}`
- `message`: `{{ $json.rendered_message }}`

**Retry configuration:** `retryOnFail: true`, `maxTries: 2`, `waitBetweenTries: 30000` (30 seconds)

**On error (second failure):** Connect to `Log 24h Failure` node.

### Task 8: Mark 24h reminder sent
**Node name:** `Mark 24h Sent`  
**Type:** `n8n-nodes-base.postgres`  
**Operation:** executeQuery

SQL:
```sql
UPDATE appointments
SET reminder_24h_sent = true, updated_at = now()
WHERE id = '{{ $json.appointment_id }}'::uuid
```

### Task 9: Log 24h failure node
**Node name:** `Log 24h Failure`  
**Type:** `n8n-nodes-base.postgres`  
**Operation:** executeQuery

SQL:
```sql
INSERT INTO workflow_errors (workflow_name, node_name, error_message, error_details)
VALUES (
  'appointment-reminders',
  'Send 24h WA Reminder',
  'WhatsApp send failed after retry for appointment {{ $json.appointment_id }}',
  '{{ JSON.stringify({ appointment_id: $json.appointment_id, phone: $json.phone }) }}'::jsonb
)
```

After logging, still mark reminder sent to avoid infinite loops:
```sql
UPDATE appointments SET reminder_24h_sent = true, updated_at = now()
WHERE id = '{{ $json.appointment_id }}'::uuid
```

(Use a separate sequential Postgres node `Mark 24h Sent After Failure`.)

### Task 10: Mirror tasks 6–9 for 1h reminder

### Task 11: Wire the complete flow
Connection order:
```
Schedule Trigger
  → Fetch Reminder Template
  → [If template exists?]
     No → Log No Template Error (INSERT workflow_errors) → [stop]
     Yes → Query 24h Appointments + Query 1h Appointments (parallel)
  
Query 24h Appointments → Loop 24h Items
  → Render 24h Message → Send 24h WA Reminder
      success → Mark 24h Sent
      failure → Log 24h Failure → Mark 24h Sent After Failure

Query 1h Appointments → Loop 1h Items
  → Render 1h Message → Send 1h WA Reminder
      success → Mark 1h Sent
      failure → Log 1h Failure → Mark 1h Sent After Failure
```
