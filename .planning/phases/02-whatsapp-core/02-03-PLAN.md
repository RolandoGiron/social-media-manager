**Workflow 1: `whatsapp-connection-update.json`** (per D-10)

Create a valid n8n workflow JSON with these nodes:

1. **Webhook node** ("Receive CONNECTION_UPDATE"):
   - Method: POST
   - Path: `whatsapp-inbound/connection-update` (per Pitfall 3: Evolution API transforms CONNECTION_UPDATE to /connection-update)
   - Response mode: Immediately (200 OK)

2. **IF node** ("Is Disconnect?"):
   - Condition: `{{ $json.data.state }}` equals `"close"` OR equals `"connecting"`
   - True branch: proceed to Send Alert
   - False branch: proceed to Log Reconnection

3. **HTTP Request node** ("Send Alert to Admin") -- True branch (per D-06, D-07):
   - Method: POST
   - URL: `http://evolution-api:8080/message/sendText/{{ $env.EVOLUTION_INSTANCE_NAME || 'clinic-main' }}`
   - Authentication: Header Auth with name `apikey` and value `{{ $env.EVOLUTION_API_KEY }}`
   - Body (JSON):
     ```json
     {
       "number": "{{ $env.ADMIN_WHATSAPP_NUMBER }}",
       "text": "⚠️ La sesion de WhatsApp se desconecto a las {{ $now.format('yyyy-MM-dd HH:mm') }}.\n\nEscanea el QR para reconectar:\nhttps://{{ $env.ADMIN_SUBDOMAIN }}.{{ $env.DOMAIN }}/WhatsApp\n\nEstado reportado: {{ $json.data.state }}"
     }
     ```
   - On Error: Continue (per D-09 -- if clinic number is disconnected, alert fails silently)
   - Retry on Fail: false (per D-08 -- alert fires once per event)

4. **NoOp node** ("Log Reconnection") -- False branch:
   - Placeholder for future logging. In Phase 7, this can write to `workflow_errors` table.
   - For now, just a No Operation node so the workflow doesn't error on `state: open` events.

**Important n8n JSON structure notes:**
- The workflow JSON must include `"name"`, `"nodes"` array, `"connections"` object, and `"active": true`
- Each node needs `"id"`, `"name"`, `"type"`, `"typeVersion"`, `"position"`, and `"parameters"`
- Webhook node type: `"n8n-nodes-base.webhook"`
- IF node type: `"n8n-nodes-base.if"`
- HTTP Request node type: `"n8n-nodes-base.httpRequest"`
- NoOp node type: `"n8n-nodes-base.noOp"`
- Use n8n expression syntax `={{ $json.data.state }}` (with `=` prefix for expressions)
- For env vars in n8n: `={{ $env.VARIABLE_NAME }}`

**n8n environment variables:** The n8n container needs access to `EVOLUTION_API_KEY`, `EVOLUTION_INSTANCE_NAME`, `ADMIN_WHATSAPP_NUMBER`, `DOMAIN`, and `ADMIN_SUBDOMAIN`. These must be added to the n8n service in docker-compose.yml. Add them:

```yaml
# Add to n8n service environment: in docker-compose.yml
      EVOLUTION_API_KEY: ${EVOLUTION_API_KEY}
      EVOLUTION_INSTANCE_NAME: ${EVOLUTION_INSTANCE_NAME:-clinic-main}
      ADMIN_WHATSAPP_NUMBER: ${ADMIN_WHATSAPP_NUMBER}
      DOMAIN: ${DOMAIN}
      ADMIN_SUBDOMAIN: ${ADMIN_SUBDOMAIN}
```
