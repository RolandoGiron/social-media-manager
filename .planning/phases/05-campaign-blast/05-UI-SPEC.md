## Spacing Scale

Streamlit manages layout spacing internally via its column/container system. The following logical spacing tokens map to Streamlit primitives:

| Token | Value | Streamlit Primitive | Usage |
|-------|-------|---------------------|-------|
| xs | 4px | `st.caption` margin | Icon gaps, inline labels |
| sm | 8px | Default widget gap | Compact element spacing |
| md | 16px | `st.columns` gap default | Default element spacing |
| lg | 24px | Section break via `st.markdown("---")` | Section padding between step 1 and step 2 |
| xl | 32px | Empty vertical space via `st.write("")` | Between progress bar and history table |
| 2xl | 48px | Not used directly | — |
| 3xl | 64px | Not used directly | — |

Exceptions: Touch targets for "Confirmar y enviar" and "Cancelar campaña" buttons use `use_container_width=True` to maximize tap area — consistent with existing pages (Pacientes, Plantillas).

**Source:** Pattern from existing pages — all buttons in list/action views use `use_container_width=True`.

Streamlit renders these roles with the following native widget mappings:

| Role | Streamlit Widget | Approx Size | Weight | Line Height |

**Source:** Pattern from `3_Pacientes.py`, `5_Inbox.py`, `4_Plantillas.py` — all use title/subheader/write/caption hierarchy.

## Color

Streamlit's default color system provides all semantic colors needed. No custom hex values are introduced.

| Role | Streamlit Semantic | Usage |
|------|-------------------|-------|
| Dominant (60%) | `background-color` (white/light grey auto) | Page background, widget surfaces |
| Secondary (30%) | Streamlit sidebar + card containers | Sidebar nav, `st.expander`, `st.container` backgrounds |
| Accent (10%) | Primary blue (`#FF4B4B` Streamlit red as CTA) | "Confirmar y enviar" primary button only |
| Destructive | `st.error` red / `type="primary"` on cancel | "Cancelar campaña" button, `st.error()` messages |

Accent reserved for:
- The single "Confirmar y enviar" button in the confirmation gate (D-04)
- Progress bar fill via `st.progress()` (Streamlit renders this in accent blue automatically)

Destructive reserved for:
- "Cancelar campaña" button (visible only while status = `in_progress`) — render as `st.button("Cancelar campaña", type="secondary")` with a preceding `st.warning()` or inline red label
- `failed_count` displayed in `st.error()` callout below the progress bar when failed_count > 0 (Claude's discretion — D ref: "error state visual treatment in progress bar shown in red")

**Source:** CONTEXT.md Claude's Discretion item on failed_count red treatment; existing pages use `st.error` / `st.success` / `st.info` for semantic color — no custom CSS.

## Page Layout: 7_Campañas.py

Three mutually exclusive views controlled by `session_state.campanas_mode` (values: `"setup"`, `"progress"`, `"history"`). Default: `"setup"`.

### View 1 — Setup (campanas_mode = "setup")

```
[Page title: Campañas]

[st.subheader: "Configurar campaña"]

Step 1 — Selección
  [st.multiselect: "Segmento (etiquetas)"]      ← reuse 3_Pacientes.py tag multiselect pattern
  [st.selectbox: "Plantilla de mensaje"]         ← reuse 4_Plantillas.py template dropdown pattern
  [st.caption: "X pacientes en este segmento"]  ← live count, updates on selection change

Step 2 — Vista previa del mensaje
  [st.subheader: "Vista previa"]
  [st.text_area (disabled): rendered message with sample values]  ← reuse templates.render_preview()

st.markdown("---")

Confirmation gate (WA-03)
  [st.markdown: "Estás a punto de enviar a **X pacientes**. ¿Confirmar?"]
  [col1: st.button("Confirmar y enviar", type="primary", use_container_width=True)]
  [col2: st.button("Cancelar", use_container_width=True)]
```

Disabled state: "Confirmar y enviar" button is `disabled=True` when no segment selected OR no template selected OR recipient count = 0.

### View 2 — Progress (campanas_mode = "progress")

```
[st.subheader: "Enviando campaña"]
[st.caption: campaign auto-name: "{tag_name} · {dd mmm yyyy}"]

[st.progress(sent_count / total_recipients)]
[st.markdown: "Enviando... X / N mensajes"]

[if failed_count > 0:]
  [st.error: "X mensajes fallaron. Continúa enviando a los destinatarios restantes."]

[if status == "in_progress":]
  [st.button("Cancelar campaña", use_container_width=True)]  ← sets status='cancelled'

[if status == "completed":]
  [st.success: "Campaña completada. X mensajes enviados."]
  [st.button("Nueva campaña", use_container_width=True)]     ← resets to setup view

[if status == "cancelled":]
  [st.warning: "Campaña cancelada. X mensajes enviados antes de cancelar."]
  [st.button("Nueva campaña", use_container_width=True)]

[auto-refresh: st_autorefresh(interval=5000) while status == "in_progress"]
```

### View 3 — History (below progress view, always visible after first campaign)

```
st.markdown("---")
[st.subheader: "Historial de campañas"]

[st.dataframe: campaigns ordered by created_at DESC]
  Columns: Nombre | Segmento | Enviados | Fallidos | Estado | Fecha
```

## Interaction Contract

### Step Flow

1. Admin lands on `"setup"` view.
2. Selects tags → recipient count updates immediately via `st.rerun()` on widget change.
3. Selects template → preview renders.
4. Clicks "Confirmar y enviar":
   - INSERT into `campaign_log` + `campaign_recipients`
   - POST to n8n webhook with `{"campaign_id": "uuid"}`
   - Set `campanas_mode = "progress"`, `st.rerun()`
5. Progress view polls every 5 seconds. Stops polling when status != `"in_progress"`.
6. Cancel: UPDATE `campaign_log.status = 'cancelled'`, `cancelled_at = now()`, `st.rerun()`.

### Button States

| Button | Visible When | Disabled When |
|--------|-------------|---------------|
| "Confirmar y enviar" | campanas_mode = "setup" | no segment, no template, or count = 0 |
| "Cancelar" (setup) | campanas_mode = "setup" | never disabled |
| "Cancelar campaña" | status = "in_progress" | never disabled (always actionable) |
| "Nueva campaña" | status = "completed" or "cancelled" | never disabled |

### Error States

| Trigger | Display |
|---------|---------|
| No recipients in segment | `st.warning("No hay pacientes en el segmento seleccionado.")` — "Confirmar" stays disabled |
| n8n webhook POST fails | `st.error("Error al iniciar la campaña. Verifica que n8n esté activo e intenta de nuevo.")` — stay in setup view |
| DB insert fails | `st.error("Error al guardar la campaña. Intenta de nuevo.")` — stay in setup view |
| failed_count > 0 during send | `st.error("X mensajes fallaron.")` — below progress bar, does not stop send |
| Campaign stuck in "pending" (n8n unreachable) | `st.warning("La campaña está en cola. n8n puede estar procesando o no responde.")` — shown when status = "pending" and elapsed > 30s |
