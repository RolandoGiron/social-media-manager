## Color

Streamlit's default semantic colors only. No custom hex values.

| Role | Streamlit Semantic | Usage |
|------|-------------------|-------|
| Dominant (60%) | `background-color` (white/light grey auto) | Page background, widget surfaces, composer area |
| Secondary (30%) | Streamlit sidebar + `st.container` borders | Sidebar nav, image preview container, list table background |
| Accent (10%) | `type="primary"` red (Streamlit `#FF4B4B`) | Single primary CTA per view: "Programar publicación" (8_Publicaciones.py composer) and "Lanzar campaña" (7_Campañas.py — already accent in Phase 5) |
| Destructive | `st.error` red | Error rows in publication list, failed publish callouts |

Accent reserved for:
- "Programar publicación" button in `8_Publicaciones.py` composer (single primary CTA per view)
- "Lanzar campaña" button in `7_Campañas.py` (already declared in Phase 5, unchanged)
- The Step 3 checkbox does NOT consume the accent budget — it is a neutral toggle, not a CTA

Destructive reserved for:
- `st.error()` row when a `scheduled_posts.status = 'error'` appears in the list, with `error_message` displayed
- `st.error()` callout if image upload fails or file size exceeds limit
- "Eliminar publicación" action (only on `pending` posts) — rendered as `st.button("Eliminar", type="secondary")` followed by inline `st.warning` (no modal — Streamlit limitation, see Copywriting Contract)

Status color mapping in publication list (text only, no colored badges):
- `pending` → neutral text "Pendiente"
- `published` → neutral text "Publicado" (success surfaced via `st.success` callout when state changes, not in list cell)
- `error` → neutral text "Error" + the row triggers an `st.error()` expander below the table showing details

**Source:** CONTEXT.md D-03 (status enum), D-09 (list columns), D-15 (n8n updates status), Claude's Discretion (image preview, error handling).

## Page Layout

This phase introduces **one new page** (`8_Publicaciones.py`) and **extends one existing page** (`7_Campañas.py`).

### 8_Publicaciones.py — Standalone Social Publishing

Two stacked sections, both visible on initial load. No mode toggling.

```
[Sidebar via render_sidebar()]

[st.title("Publicaciones")]

[st.subheader("Crear publicación")]
  [st.text_area: "Caption", height=120, max_chars=2200]   ← Instagram limit
  [st.file_uploader: "Imagen", type=["jpg","jpeg","png","webp"]]
  [if uploaded: st.image(file, width=300)]                ← preview, Claude's discretion D-12
  [st.multiselect: "Plataformas", options=["Instagram","Facebook"], default=["Instagram","Facebook"]]
  [col1, col2 = st.columns(2)]
    [col1: st.date_input: "Fecha de publicación", min_value=today]
    [col2: st.time_input: "Hora de publicación"]
  [st.button("Programar publicación", type="primary", use_container_width=True)]
    ← disabled when caption empty OR no image OR no platform OR datetime <= now

st.markdown("---")

[st.subheader("Publicaciones programadas")]
  [if no posts: st.info("Sin publicaciones programadas. Crea tu primera publicación arriba.")]
  [else: st.dataframe(use_container_width=True, hide_index=True)]
    Columns: Caption (truncado 60 chars) | Plataformas | Fecha programada | Estado | Acciones
    Order: scheduled_at ASC (próximas primero, per D-09)

  [for each row with status == "error":]
    [st.error(f"Error en '{caption_truncated}': {error_message}")]
```

### 7_Campañas.py — Step 3 Extension (new section, additive)

Inserted **after** the existing confirmation gate (Step 2) in `setup` mode, **before** the "Confirmar y enviar" button. The current Phase 5 button is renamed to "Lanzar campaña" to reflect the unified semantics — copy change is the only edit to existing UI strings.

```
[... existing Step 1 (segment + template) ...]
[... existing Step 2 (preview + recipient count) ...]

st.markdown("---")

[st.subheader("Paso 3 — Publicar también en redes (opcional)")]
[st.checkbox("Publicar en redes sociales junto con la campaña WhatsApp", key="campanas_publish_social")]

[if campanas_publish_social:]
  [st.text_area: "Caption del post", height=120, max_chars=2200,
    help="Puede ser distinto al mensaje de WhatsApp"]
  [st.file_uploader: "Imagen", type=["jpg","jpeg","png","webp"]]
  [if uploaded: st.image(file, width=300)]
  [st.multiselect: "Plataformas", options=["Instagram","Facebook"], default=["Instagram","Facebook"]]
  [col1, col2 = st.columns(2)]
    [col1: st.date_input: "Fecha de publicación", min_value=today, value=today]
    [col2: st.time_input: "Hora de publicación", value=now+15min]

st.markdown("---")

[Confirmation gate — existing copy from Phase 5]
[st.markdown: "Estás a punto de enviar a **N pacientes**" + (if checkbox: " y publicar en redes sociales") + ". ¿Confirmar?"]
[col1: st.button("Lanzar campaña", type="primary", use_container_width=True)]
[col2: st.button("Cancelar", use_container_width=True)]
```

**Step 3 disabled state:** "Lanzar campaña" button disabled when checkbox is active AND (caption empty OR no image OR no platform OR datetime invalid). When checkbox is OFF, the existing Phase 5 disabled rules apply unchanged.

## Interaction Contract

### 8_Publicaciones.py Flow

1. Admin lands on page → composer + list visible.
2. Admin fills caption, uploads image, selects platforms and date/time.
3. Image preview renders inline (`st.image`) immediately on upload.
4. Click "Programar publicación":
   - Save image to `/opt/clinic-crm/uploads/{uuid4}.{ext}` (D-13)
   - INSERT into `scheduled_posts` with `status='pending'`
   - POST to n8n webhook `social-publish` with `{"post_id": "uuid"}` (D-15)
   - On success: `st.success("Publicación programada para {fecha}.")` + `st.rerun()` to refresh list
   - On failure: stay on form, show `st.error(...)`, do NOT clear inputs
5. List re-renders showing the new row with `status='pending'`.
6. The n8n cron `social-scheduler` (D-16) flips `status` to `published` or `error` at `scheduled_at`. Admin sees updated status on next page visit / refresh.

### 7_Campañas.py Step 3 Flow

1. Admin completes Steps 1 and 2 (Phase 5 unchanged).
2. Admin checks "Publicar en redes sociales también".
3. Composer expands inline. All Step 3 fields are **required** if checkbox is on.
4. Click "Lanzar campaña":
   - Always: existing Phase 5 path (insert `campaign_log` + `campaign_recipients` + n8n webhook for WA blast)
   - If Step 3 active: additionally save image, INSERT into `scheduled_posts`, POST to n8n `social-publish` webhook (D-06)
   - Both inserts wrapped in a single try/except — if either fails, show `st.error` and do not switch to progress view
5. On success: `campanas_mode = "progress"`, `st.rerun()` (Phase 5 behavior unchanged).
6. Admin can navigate to `8_Publicaciones.py` to monitor the social side; Phase 5 progress view monitors the WA side.

### Polling

`8_Publicaciones.py` does **not** use `streamlit_autorefresh`. Status updates require manual page refresh or sidebar navigation. Rationale: most posts are scheduled hours/days ahead, so live polling adds load without user benefit. (Live polling stays in `7_Campañas.py` progress view as defined in Phase 5.)

### Button States

| Button | Visible When | Disabled When |
|--------|-------------|---------------|
| "Programar publicación" (8_Publicaciones.py) | always | caption empty OR no image OR no platform OR scheduled_at <= now |
| "Lanzar campaña" (7_Campañas.py) — checkbox OFF | setup mode | Phase 5 rules (no segment / no template / count = 0) |
| "Lanzar campaña" (7_Campañas.py) — checkbox ON | setup mode | Phase 5 rules OR (caption empty / no image / no platform / scheduled_at <= now) |
| "Eliminar" (per row in list) | only when row.status = "pending" | never disabled |

### Error States

| Trigger | Display |
|---------|---------|
| Image upload exceeds 8 MB | `st.error("La imagen excede 8 MB. Usa una imagen más pequeña.")` |
| Unsupported image format | `st.error("Formato no soportado. Usa JPG, PNG o WEBP.")` |
| `scheduled_at` in the past | "Programar publicación" stays disabled; `st.caption("La fecha debe ser futura.")` below the time input |
| n8n webhook POST fails | `st.error("Error al programar la publicación. Verifica que n8n esté activo e intenta de nuevo.")` — keep form values |
| DB insert fails | `st.error("Error al guardar la publicación. Intenta de nuevo.")` — keep form values |
| Row.status = "error" in list | Inline `st.error(f"Error en '{caption}': {error_message}")` below the dataframe |
| MOCK_SOCIAL=true (mock mode active) | `st.info("Modo prueba activo (MOCK_SOCIAL=true). Las publicaciones se simulan sin llamar a Meta API.")` shown once at top of `8_Publicaciones.py` |
| Empty list | `st.info("Sin publicaciones programadas. Crea tu primera publicación arriba.")` |
