# Phase 3 — UI Design Contract

> Visual and interaction contract for the CRM Core admin UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Streamlit native) |
| Preset | not applicable |
| Component library | Streamlit 1.35+ built-in widgets |
| Icon library | Material Icons (via Streamlit `icon=":material/..."` syntax) |
| Font | Streamlit default (Source Sans Pro) |

**Note:** Streamlit controls all layout, spacing, typography, and color at the framework level. This contract specifies semantic design decisions and interaction patterns rather than raw CSS tokens. Spacing, typography, and color sections below define the *intent* that maps to Streamlit widget parameters.

---

## Spacing Scale

Streamlit does not expose pixel-level spacing control. Layout is managed through columns, containers, expanders, and vertical whitespace (`st.markdown("---")`).

| Pattern | Streamlit Mechanism | Usage |
|---------|---------------------|-------|
| Inline gap | `st.columns([ratio, ratio])` with gap parameter | Side-by-side elements (filters, buttons) |
| Section break | `st.markdown("---")` or `st.divider()` | Between filter bar and data table, between sections |
| Compact group | `st.container()` | Grouping related controls (search + filters) |
| Expanded detail | `st.expander()` | Tag creation form, import preview |
| Page padding | `layout="wide"` (inherited from app.py) | Full-width content area |

Exceptions: none

---

## Typography

Streamlit controls font rendering. These roles map to specific Streamlit API calls:

| Role | Streamlit API | Usage |
|------|---------------|-------|
| Page title | `st.title("text")` | One per page: "Pacientes", "Plantillas de Mensaje" |
| Section heading | `st.subheader("text")` | Import preview header, template list header |
| Body text | `st.markdown("text")` or `st.text("text")` | Descriptions, instructions, status messages |
| Caption / metadata | `st.caption("text")` | Row counts ("Mostrando 1-25 de 142 pacientes"), timestamps |
| Status badge | `st.markdown(":green[text]")` | Import status indicators |

**Line height:** Body text line height is Streamlit's default (~1.5). No override is applied. This is an intentional decision -- Streamlit's built-in line height provides adequate readability for data-heavy admin screens.

## Color

Streamlit uses its own theme. This contract defines semantic color usage through Streamlit's built-in status components and markdown color syntax.

| Role | Mechanism | Usage |
|------|-----------|-------|
| Success (green) | `st.success()` / `:green[text]` / `:green_circle:` | Import complete confirmation, "Nuevo" badge in CSV preview |
| Warning (amber) | `st.warning()` / `:orange[text]` | "Duplicado" badge in CSV preview, tag delete blocked message |
| Error (red) | `st.error()` / `:red[text]` | CSV parse errors, "Error" badge in CSV preview, validation failures |
| Info (blue) | `st.info()` | Empty states, instructional messages |
| Tag colors | `st.markdown()` with inline HTML color from `tags.color` column | Tag chips in patient list (default: `#6366f1` indigo) |

**60/30/10 color distribution:** Dominant (60%): Streamlit default surface (white in light mode, dark grey in dark mode) -- all page backgrounds and content areas. Secondary (30%): sidebar navigation background and card-like containers (`st.container`, `st.expander`). Accent (10%): `st.success`, `st.warning`, `st.error`, and `st.info` status colors, used sparingly and only for actionable feedback (import results, validation errors, confirmation messages). Tag color chips in the patient list are the only non-status use of color.

**Import status badges (D-01):**

| Badge | Display | Color mechanism |
|-------|---------|-----------------|
| Nuevo | `:green[` `check` `Nuevo]` | Green markdown |
| Duplicado | `:orange[` `warning` `Duplicado]` | Orange markdown |
| Error | `:red[` `x` `Error]` | Red markdown |

## Component Inventory

### Page: Pacientes (`pages/3_Pacientes.py`)

**Focal point:** Primary focal point is the paginated patient data table -- this is the element the user's eye should land on first. Secondary anchor is the "Importar pacientes" CTA button in the header row, providing the primary action entry point.

**Layout structure (top to bottom):**

1. `st.title("Pacientes")` + `st.button("Importar pacientes", icon=":material/upload_file:")` in a 2-column header row (title left, button right)
2. **Filter bar** (when not in import mode): `st.text_input` for search + `st.multiselect` for tag filter + tag management expander, all in a single row using `st.columns([3, 2, 1])`
3. **Patient data table**: `st.dataframe` with `on_select="rerun"` and `selection_mode="multi-row"` for checkbox selection
4. **Bulk action bar** (visible when rows selected): `st.multiselect` for tag selection + `st.button("Asignar etiquetas")` in a container below the table
5. **Pagination controls**: Previous/Next buttons + page caption in a bottom row
6. **Import flow** (toggled by CTA button, replaces filter+table): file uploader, preview table, summary, confirm/cancel buttons

**States:**

| State | What renders |
|-------|-------------|
| Default (has patients) | Filter bar + paginated data table + page caption |
| Empty (no patients) | Empty state message + prominent import CTA |
| Import: file selection | `st.file_uploader` accepting .csv, .xlsx, .xls |
| Import: preview | Preview `st.dataframe` with Estado column + summary row + "Importar X nuevos" / "Descartar importacion" buttons |
| Import: processing | `st.spinner("Importando pacientes...")` |
| Import: complete | `st.success` with counts + auto-return to patient list |
| Rows selected | Bulk action bar appears below table |
| Search active | Table filtered by name/phone ILIKE query |
| Tag filter active | Table filtered by tag JOIN |

### Tag Management (inline in Pacientes page)

**Location:** `st.expander("Nueva etiqueta")` inside the filter bar area.

| Element | Widget | Details |
|---------|--------|---------|
| Tag name | `st.text_input("Nombre de etiqueta")` | Required, max 50 chars |
| Tag color | `st.color_picker("Color", value="#6366f1")` | Default indigo |
| Create button | `st.button("Crear etiqueta")` | Disabled until name is non-empty |
| Existing tags list | `st.markdown` with colored chips | Each tag shows name + patient count + delete icon |
| Delete blocked | `st.warning("Esta etiqueta esta asignada a N pacientes. Elimina las asignaciones primero.")` | Per D-08 |

### Page: Plantillas (`pages/4_Plantillas.py`)

**Layout structure (top to bottom):**

1. `st.title("Plantillas de Mensaje")`
2. **Template list** (left state) OR **Editor view** (when creating/editing)
3. Template list: `st.dataframe` with columns [Nombre, Categoria, Variables, Creada] + "Nueva plantilla" button

**Editor layout (D-09 side-by-side):**

Using `st.columns([1, 1])`:

| Left column (Editor) | Right column (Preview) |
|----------------------|----------------------|
| `st.text_input("Nombre de plantilla")` | `st.subheader("Vista previa")` |
| `st.selectbox("Categoria", ["general", "promocion", "recordatorio"])` | Rendered message body with sample values |
| `st.text_area("Cuerpo del mensaje", height=300)` | Variables detected shown as chips below preview |
| `st.button("Guardar plantilla")` / `st.button("Descartar cambios")` | |

**States:**

| State | What renders |
|-------|-------------|
| Default (has templates) | Template list table + "Nueva plantilla" button |
| Empty (no templates) | Empty state message + "Nueva plantilla" CTA |
| Editing | Side-by-side editor + preview (replaces list) |
| Save success | `st.success` + return to list |
| Validation error | `st.error` inline (missing name or empty body) |

## Interaction Contracts

### CSV Import Flow

```
[Patient list page]
  |
  v  click "Importar pacientes"
[File uploader visible, table hidden]
  |
  v  user selects .csv or .xlsx
[st.spinner while parsing]
  |
  v  parsing complete
[Preview table with Estado column]
[Summary: "X nuevos / Y duplicados / Z errores"]
[Buttons: "Importar X nuevos" | "Descartar importacion"]
  |
  |-- click "Descartar importacion" --> [Back to patient list]
  |-- click "Importar X nuevos" --> [st.spinner("Importando...")]
                                      |
                                      v
                                    [st.success("Se importaron N pacientes")]
                                    [Auto-rerun: patient list refreshes with new data]
```

### Patient Search + Filter

- Search is debounced by Streamlit's natural rerun-on-enter behavior
- Tag multiselect filter applies immediately on change (Streamlit default)
- Search and tag filters combine with AND logic: patients must match search text AND have at least one selected tag
- Clearing all filters shows full paginated list

### Bulk Tag Assignment

```
[Select rows via checkbox in st.dataframe]
  |
  v  selection triggers rerun
[Bulk action bar appears: tag multiselect + "Asignar etiquetas" button]
  |
  v  click "Asignar etiquetas"
[st.spinner("Asignando etiquetas...")]
  |
  v
[st.success("Etiquetas asignadas a N pacientes.")]
[Selection cleared, table refreshes]
```

### Template Editor Save

```
[Editor view: name + category + body fields, live preview on right]
  |
  v  click "Guardar plantilla"
[Validate: name non-empty, body non-empty]
  |
  |-- validation fails --> [st.error with specific message, no navigation]
  |-- validation passes --> [Extract {{variables}} from body]
                              |
                              v
                            [INSERT/UPDATE to message_templates]
                              |
                              v
                            [st.success + return to template list]

[Editor view]
  |
  v  click "Descartar cambios"
[Return to template list without saving]
```

## Data Table Column Specifications

### Patient List (`st.dataframe`)

| Column | Source | Width | Notes |
|--------|--------|-------|-------|
| Nombre | `first_name \|\| ' ' \|\| last_name` | auto | Primary identifier |
| Telefono | `phone_normalized` | 160px | Formatted +52 number |
| Etiquetas | JOIN `patient_tags` + `tags` | auto | Colored tag names, comma-separated |
| Fuente | `source` | 100px | "csv_import" or future sources |
| Registrado | `created_at` | 120px | Date only, format: "01 abr 2026" |

### CSV Import Preview (`st.dataframe`)

| Column | Source | Notes |
|--------|--------|-------|
| Nombre | Parsed from CSV | As provided |
| Apellido | Parsed from CSV | As provided |
| Telefono | Parsed from CSV | Original value |
| Tel. Normalizado | Computed | +52 format applied |
| Estado | Computed | Badge: Nuevo / Duplicado / Error |

### Template List (`st.dataframe`)

| Column | Source | Notes |
|--------|--------|-------|
| Nombre | `name` | Template identifier |
| Categoria | `category` | "general" / "promocion" / "recordatorio" |
| Variables | `variables` array | Comma-separated list |
| Creada | `created_at` | Date format: "01 abr 2026" |

## Destructive Actions

| Action | Confirmation Approach | Copy |
|--------|----------------------|------|
