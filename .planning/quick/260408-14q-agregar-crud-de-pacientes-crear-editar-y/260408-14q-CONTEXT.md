# Quick Task 260408-14q: CRUD de pacientes - Context

**Gathered:** 2026-04-08
**Status:** Ready for planning

<domain>
## Task Boundary

Agregar operaciones CRUD en la página Pacientes (3_Pacientes.py): crear paciente manualmente, editar datos de un paciente existente, y borrar paciente(s) desde la grilla.

</domain>

<decisions>
## Implementation Decisions

### Borrar pacientes
- Opción C: doble mecanismo
  - Bulk: seleccionar una o varias filas → botón "Borrar seleccionados" en la barra de acciones (junto al de asignar etiquetas), con diálogo de confirmación
  - Individual: ícono de borrar (🗑) en cada fila de la tabla

### Formulario crear/editar
- Sección expandible (st.expander) debajo de la tabla
- Para crear: expander "Nuevo paciente" siempre visible
- Para editar: al hacer click en una fila, el expander se puebla con los datos del paciente seleccionado

### Campos del formulario
- Completo: nombre, apellido, teléfono, notas (enfermedad), email, fuente
- Todos opcionales excepto nombre, apellido y teléfono

### Claude's Discretion
- Validación de teléfono al crear/editar usa la misma función normalize_sv_phone existente
- Borrado elimina también patient_tags asociadas (CASCADE en DB o DELETE explícito)
- Confirmación de borrado bulk: st.warning con conteo ("¿Borrar N pacientes? Esta acción no se puede deshacer.")

</decisions>
